# CLAWOS 详细开发步骤文档（基于 `ref-data/need.md` V1.0）

## 1. 文档目标与范围
本文件用于把需求文档转为可执行的研发计划，覆盖以下子系统：

- 树莓派固件端（`firmware`）
- 客户端（手机/桌面，建议统一核心代码，目录可落在 `client`）
- 云端中转与账号服务（建议落在 `app` 或 `web` 的服务子目录）
- 运维与发布体系（CI/CD、监控、日志与审计）

本计划默认优先级：
- 第一优先：安全模型正确（E2EE、密钥不上公网）
- 第二优先：核心链路可用（配网、远控、群控）
- 第三优先：体验优化（文件权限分级、命令历史、快捷操作）

---

## 2. 总体实施策略
采用“三层并行、里程碑收敛”方式：

- 纵向三层：固件层、云服务层、客户端层并行开发
- 横向里程碑：每个阶段都要求端到端联调通过，再进入下一阶段
- 安全门禁：每阶段增加安全验收条款，未通过不得进入生产阶段

建议节奏（10~14 周，按 2 周一个迭代）：

1. `Sprint 1-2`：账号、会话链接、基础消息中转打通
2. `Sprint 3-4`：蓝牙首次配网 + 密钥分发打通
3. `Sprint 5-6`：cmd/nl 双通道与多用户共享
4. `Sprint 7`：文件权限分级与局域网同步
5. `Sprint 8`：安全加固、压测、灰度发布

## 4. 关键技术决策（必须先冻结）
在正式编码前完成以下决策并形成 ADR（Architecture Decision Record）：

1. 密码方案
- 消息加密算法：`X25519 + XChaCha20-Poly1305` 或 `ECDH + AES-256-GCM`
- 密钥派生：`HKDF-SHA256`
- 消息签名：`Ed25519`
- 重放防护：`nonce + timestamp + message_id`

2. 会话链接模型
- `session_link` 为永久唯一标识（建议 UUIDv7 + 短链映射）
- 云端只存密文与元数据，不存任何明文密钥

3. 通信机制
- 全链路 HTTP(S) 短连接 + 轮询（符合“无长连接”要求）
- 拉取接口支持 `cursor` 增量同步和离线补偿

4. 本地存储与密钥保管
- 客户端密钥存设备安全区（Keychain/Keystore/DPAPI）
- 树莓派使用文件系统加密区 + 最小权限访问

5. 局域网直连策略
- 局域网发现：mDNS 或管理员配置直连地址
- 传输协议：HTTPS + 双向认证（设备证书）

## 5. 分阶段开发步骤

## 阶段 0：云端基础服务（第 2~3 周）
目标：实现账号、设备、会话链接、消息中转最小闭环。

开发步骤：
1. 建立认证服务
- 注册/登录/刷新 token
- 用户 ID 全局唯一

2. 建立设备与会话服务
- 一台设备仅一个永久会话链接
- 支持一个账号绑定多设备
- 支持多用户绑定同设备

3. 建立消息中转服务
- `POST /relay/send`：发送密文
- `GET /relay/pull`：按会话+游标拉取
- 离线缓存与 TTL 清理

4. 建立权限服务
- 角色：`admin/member`
- 用户-设备-会话映射表

5. 建立审计日志（仅元数据）
- 谁在何时对哪个会话做了何操作
- 不记录明文内容

数据库最小表：
- `users`
- `devices`
- `session_links`
- `user_device_roles`
- `relay_messages`
- `audit_logs`

验收标准：
- Postman/自动化测试可完成：注册 -> 登录 -> 绑定设备 -> 发消息 -> 拉消息
- 数据库中不可出现密钥字段和明文字段

---

## 阶段 1：树莓派基础 Agent（第 3~4 周）
目标：树莓派可轮询云端、解密、执行并回传结果。

开发步骤：
1. 开发设备 Agent 常驻进程
- 开机自启动（systemd）
- 定时轮询云端会话链接
- 本地消息队列与重试机制

2. 开发消息处理流水线
- 验签 -> 解密 -> 分流（cmd/nl）
- 执行 -> 输出收集 -> 加密回传

3. 开发 cmd 执行器
- 命令白名单与危险命令拦截
- 超时控制与资源限制（CPU/内存）

4. 接入 OpenClaw/nl 执行器
- nl 消息进入 OpenClaw
- 统一回传格式

5. 落地本地安全存储
- 会话链接、WiFi、密钥分别存放
- 文件权限 `600` + 运行用户最小化

验收标准：
- 云端发送密文命令后，设备可执行并返回密文结果
- 未授权命令被拦截且可审计

---

## 阶段 2：蓝牙首次配网与密钥下发（第 4~5 周）
目标：完成“唯一一次蓝牙配网”闭环。

开发步骤：
1. 开发 BLE 广播与握手协议
- 出厂模式广播
- 管理员身份确认（一次性配对码或短时令牌）

2. 开发配网数据包协议
- WiFi SSID/密码
- 会话链接
- 公共密钥/管理员个人密钥
- 包完整性校验与重放防护

3. 实现设备模式切换
- 配网完成后自动关闭 BLE
- 切换至公网轮询模式

4. 异常恢复机制
- WiFi 失败回滚
- 配网超时重置
- 断电续配策略

验收标准：
- 全新设备可在 5 分钟内完成首次初始化
- 成功配网后 BLE 不再可发现（除管理员触发重置）

---

## 阶段 3：客户端核心能力（第 5~7 周）
目标：客户端具备设备管理、cmd/nl 双入口、加解密与历史展示。

开发步骤：
1. 建立 `core-sdk`
- 统一协议模型
- 加解密、签名、验签
- 会话发送/拉取与游标同步

2. 实现账号与设备管理页
- 登录注册
- 设备列表、在线状态、绑定关系

3. 实现控制台（cmd）
- 命令输入/发送
- 执行结果展示
- 历史命令与快捷命令

4. 实现聊天框（nl）
- 群聊消息列表（发送者 ID/昵称）
- nl 请求与响应渲染

5. 多端一致性
- 手机与桌面共享 `core-sdk`
- 界面层仅做适配

验收标准：
- 同一账号在手机/桌面都可远程发送 cmd 与 nl
- 任何消息离开客户端前均已加密

---

## 阶段 4：多用户共享与公私空间（第 7~8 周）
目标：实现管理员分享设备、成员加入、公私空间隔离。

开发步骤：
1. 设备分享流程
- 管理员生成分享链接（短时有效 + 可撤销）
- 新用户输入链接完成绑定

2. 密钥体系落地
- 公共空间密钥（共享）
- 用户个人隐私密钥（独立）

3. 空间权限实现
- 公共空间：授权成员可读写
- 个人空间：仅本人可见
- 管理员不能读取他人私有数据

4. 角色权限控制
- admin/member 的接口级校验
- 前端按钮级权限收敛

验收标准：
- 成员加入后可用公共空间但不可见他人个人空间
- 管理员具备管理权但无越权读取能力

---

## 阶段 5：文件权限分级与局域网同步（第 8~10 周）
目标：完成 Level 0/1/2 远程可见策略与 LAN 高速同步。

开发步骤：
1. 文件元数据模型
- `space_type`（public/private）
- `privacy_level`（0/1/2）
- `owner_user_id`

2. 远程访问分级控制
- Level 0：目录结构都不可见
- Level 1：仅目录与元数据可见
- Level 2：允许有限预览（头部/缩略）

3. 局域网直连同步
- 设备发现与鉴权
- 断点续传
- 冲突处理（版本号/时间戳）

4. 文件加密策略
- 文件分块加密
- 本地密文存储
- 公网链路不传输完整原文件

验收标准：
- 公网下任意等级都无法下载完整原文件
- 局域网下可完成大文件稳定同步

---

## 阶段 6：安全加固与合规（第 10~11 周）
目标：把“可用”提升到“可上线”。

开发步骤：
1. 密钥安全加固
- 密钥轮换策略（公共密钥轮换、用户密钥吊销）
- 设备丢失/用户移除后的密钥失效机制

2. 攻击面防护
- 接口限流与防暴力破解
- 消息重放/伪造检测
- 固件端命令注入防护

3. 安全测试
- 静态扫描（SAST）
- 依赖漏洞扫描（SCA）
- 基础渗透测试（认证、鉴权、注入、越权）

4. 审计与追踪
- 全链路 trace_id
- 操作审计可导出

验收标准：
- 高危漏洞为 0
- 所有安全关键流程有自动化回归用例

---

## 阶段 7：测试、灰度与发布（第 11~12 周）
目标：分环境发布并验证稳定性。

开发步骤：
1. 建立环境
- `dev` / `staging` / `prod`
- 每环境独立密钥与配置

2. 自动化测试矩阵
- 单元测试：加解密、协议编解码、权限判定
- 集成测试：云-端-客户端链路
- E2E 测试：首次配网、远控、群控、权限分级

3. 性能与稳定性
- 云端中转吞吐压测
- 设备轮询规模压测
- 弱网与断网恢复测试

4. 灰度发布
- 按设备批次灰度（5% -> 20% -> 50% -> 100%）
- 监控指标守门（错误率、延迟、失败重试率）

验收标准：
- 连续 7 天灰度无 P0/P1 故障
- 关键接口 SLO 达标

## 6. 接口与协议落地清单（建议优先实现）
最小 API（示例）：

- `POST /auth/register`
- `POST /auth/login`
- `POST /devices/bind`
- `POST /sessions/create`
- `POST /relay/send`
- `GET /relay/pull?session_id=&cursor=`
- `POST /share/create`
- `POST /share/join`
- `GET /files/list`
- `GET /files/preview`

最小消息体（密文）：

```json
{
  "session_id": "string",
  "from_user_id": "string",
  "from_user_role": "admin|member",
  "msg_type": "cmd|nl",
  "content": "base64(ciphertext)",
  "nonce": "base64",
  "timestamp": 1710000000,
  "message_id": "uuid"
}
```

## 7. 测试与验收用例（必须覆盖）
必须有自动化回归的核心场景：

1. 首次配网成功与失败回滚
2. 远程 cmd 执行与结果回传
3. 远程 nl 群聊消息可达与顺序一致
4. 多用户加入与角色权限隔离
5. 公共/私有空间可见性校验
6. 文件权限 Level 0/1/2 行为正确
7. 局域网同步成功与断点续传
8. 密钥轮换后旧密文不可被未授权用户解密

## 8. 项目管理建议（执行层）
建议按“里程碑 + 每周例会”推进：

1. 每周固定输出
- 本周完成项
- 下周计划
- 阻塞项与决策需求

2. 风险看板（持续维护）
- BLE 兼容性风险
- 多平台密钥存储差异风险
- 局域网发现稳定性风险
- OpenClaw 推理延迟风险

3. 发布守门条件
- 功能验收通过率 >= 98%
- 安全测试高危 = 0
- 性能基线满足目标值

## 9. MVP 与后续演进边界
MVP 必做：
- 首次蓝牙配网
- 会话链接中转
- cmd/nl 双入口
- 多用户共享与角色权限
- 公私空间隔离
- 文件权限分级（0/1/2）

V1.1+ 可迭代：
- 更细粒度审计报表
- 多设备批量运维编排
- 高级局域网同步策略（去重、增量块）
- 更丰富的可观测性看板
